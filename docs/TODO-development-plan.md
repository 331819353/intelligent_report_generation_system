## 1. Codex 执行规则

### 1.1 每次领取任务前

- 阅读本文件、需求文档及仓库中的 `AGENTS.md`（若后续新增）。
- 一次只领取一个可独立验收的任务；不得把多个阶段一次性混在同一变更中。
- 检查当前工作区状态，保留用户已有修改，不覆盖无关文件。
- 若任务依赖尚未完成，先完成依赖任务或明确报告阻塞，不用临时假实现掩盖依赖。
- 涉及数据库、对象存储、消息队列、联邦查询引擎或云模型时，先提供本地可替代实现和清晰接口边界。

### 1.2 每个任务必须包含

- 实现代码；
- 单元测试或集成测试；
- 必要的数据库迁移；
- API/Schema/配置文档更新；
- 本地验证命令；
- 对安全、租户隔离和版本兼容性的检查。

### 1.3 完成定义（Definition of Done）

任务只有同时满足以下条件才能标记为 `[x]`：

1. 需求和验收条件全部实现；
2. 格式化、静态检查、单元测试和相关集成测试通过；
3. 不包含明文凭证、临时调试代码或无法解释的跳过测试；
4. 新增配置均提供默认值和示例；
5. 新增 API 有请求、响应及错误码说明；
6. 新增持久化结构有向前迁移方案；
7. 多租户数据访问包含 `tenant_id` 校验；
8. Codex 在任务交付说明中列出修改文件、验证结果、遗留风险和下一建议任务。

### 1.4 建议统一验证命令

工程初始化后应提供统一入口，目标命令如下：

```bash
make fmt
make lint
make test
make test-integration
make build
```

前端初始化后增加：

```bash
make frontend-lint
make frontend-test
make frontend-build
```
