# 数据集列转换组件产品与开发规格

版本：V1.1  
状态：实施中  
优先级：P0  
适用范围：数据集配置中心的关系组件画布

## 1. 背景

当前数据集画布已经支持数据节点、关联组件、分组组件和结束节点，可以完成字段投影、多表关联、分组聚合及最终输出配置。但业务用户仍无法在图形画布中直接完成常见的列内容加工，例如：

- 将字符串日期转换为标准日期；
- 从编码或证件号中截取部分内容；
- 将金额从分转换为元并保留两位小数；
- 将空字符串、NULL 和异常占位值统一处理；
- 根据状态码、金额区间或多个条件生成业务分类；
- 将一个字段拆成多个字段，或将多个字段拼接为一个字段。

现有 DSL 已支持 `DATE_TRUNC`、`CAST`、`ADD`、`SUBTRACT`、`MULTIPLY`、`DIVIDE`、`CONCAT`、`COALESCE` 和 `CASE` 等基础表达式，但当前关系组件画布没有把这些能力完整暴露给用户，文本截取、日期解析、日期格式化、正则提取、数值舍入和字段拆分等高频能力也尚未形成跨数据库与文件执行器一致的正式语义。

## 2. 产品目标

在现有关系组件画布中增加七类列转换能力，使具备字段业务知识但不会编写 SQL 的用户能够：

1. 通过拖拽和表单配置完成列级加工；
2. 在保存前看到输入值、输出值和转换失败情况；
3. 将转换产物继续连接关联、分组、其他字段处理或结束节点；
4. 保存后能够稳定回载、继续编辑并保持字段血缘；
5. 在 MySQL、Oracle、Excel/CSV 和跨源执行路径中获得一致结果；
6. 不通过转换组件绕过字段权限、脱敏、敏感级别和审计边界。

## 3. 非目标

P0 不包含以下能力：

- 自定义 SQL、自定义脚本或任意函数执行；
- 改变记录数量的过滤、去重、聚合、Union、行转列或列转行；
- 依赖外部服务的地址解析、自然语言识别或第三方字典查询；
- 由数据集转换组件承担报表展示格式，例如货币符号、千分位和仅用于展示的日期文案；
- 允许普通数据集编辑者通过字段转换自行配置脱敏规则；
- 在首期让数据集 AI 自动生成或修改字段处理组件。

过滤、去重、排序与 Top N 属于相邻的行级数据加工能力，应使用独立组件和明确的行数变化语义，不混入本规格。

## 4. 核心产品决策

### 4.1 组件栏展示细粒度业务卡片

组件栏不再用“文本处理”“数值处理”这类大而全入口承载多个互不相干的功能。转换能力按用途拆为独立卡片，同类组件使用同一颜色，并在分类内按中文名称拼音首字母 A–Z 排序：

| 分类 | 卡片（分类内 A–Z） | `componentType` | 当前处理逻辑 | P0 是否改变行数 |
|---|---|---|---|---|
| 文本 | 大写转换 | `TEXT_UPPER` | `UPPER` | 否 |
| 文本 | 空格清理 | `TEXT_TRIM` | `TRIM` | 否 |
| 文本 | 文本替换 | `TEXT_REPLACE` | `REPLACE` | 否 |
| 文本 | 小写转换 | `TEXT_LOWER` | `LOWER` | 否 |
| 文本 | 字段截取 | `TEXT_SUBSTRING` | `SUBSTRING` | 否 |
| 文本 | 字段拼接 | `TEXT_CONCAT` | `CONCAT` | 否 |
| 数值 | 取绝对值 | `NUMBER_ABSOLUTE` | `ABS` | 否 |
| 数值 | 数值取整 | `NUMBER_ROUNDING` | `ROUND` / `FLOOR` / `CEIL` | 否 |
| 数值 | 数值运算 | `NUMBER_ARITHMETIC` | `ADD` / `SUBTRACT` / `MULTIPLY` / `DIVIDE` | 否 |
| 日期 | 日期转换 | `DATE_FORMAT` | `DATE_FORMAT` | 否 |
| 规则 | 空值填充 | `NULL` | `COALESCE` | 否 |
| 规则 | 类型转换 | `CAST` | `CAST` | 否 |
| 规则 | 条件映射 | `CONDITION` | `CASE` | 否 |

所有公开卡片名称采用四个易理解的汉字。历史草稿中的 `TEXT`、`NUMBER` 和 `SPLIT_MERGE` 大类仍可回载和编辑，但不再作为新建入口展示。

### 4.2 底层只增加一种画布组件

细粒度卡片只提升功能可发现性，序列化和拓扑层仍使用统一的 `TRANSFORM` 组件。`family` 保留为执行与旧草稿兼容所需的基础类型，新增 `componentType` 标识用户选择的具体入口。不得为每个函数增加一种 `GraphInput.kind`。

这样可以复用同一套连线、移动、删除、回载、循环检测、字段产物和配置抽屉逻辑，并避免 `GraphInput.kind` 和数据集 AI 的组件枚举随函数数量持续膨胀。

建议的设计器结构为：

```json
{
  "id": "transform_1",
  "name": "日期转换 1",
  "family": "DATE",
  "componentType": "DATE_FORMAT",
  "input": { "kind": "NODE", "id": "node_1" },
  "position": { "x": 420, "y": 180 },
  "rules": []
}
```

### 4.3 单输入、单输出、保持行数

- 每个字段处理组件只有一个上游输入和一个产物输出；
- 输入可以来自 `NODE`、`JOIN`、`GROUP` 或另一个 `TRANSFORM`；
- 输出可以进入 `JOIN`、`GROUP`、另一个 `TRANSFORM` 或 `END`；
- P0 的每条输入记录必须恰好产生一条输出记录；
- 转换失败不得提供“丢弃该行”，需要删除异常行时应由未来的过滤组件完成；
- 同一组件中的多条规则默认并行读取上游字段，不能引用同组件内另一条规则刚生成的字段；需要连续加工时使用两个字段处理组件串联，确保执行顺序在画布中可见。

### 4.4 默认新增字段，不直接覆盖来源

默认行为是保留上游字段并新增转换字段。用户可显式选择“替换来源字段”，其实际语义为：

1. 生成一个具有独立稳定内部身份的新字段；
2. 从当前组件产物中隐藏被替换的来源字段；
3. 新字段可以继承来源字段的业务名称和对外编码；
4. 血缘仍保留“来源字段 → 转换规则 → 新字段”的完整关系。

不得通过原地修改物理字段引用的方式伪造覆盖，否则草稿回载、下游引用和字段级影响分析无法区分转换前后的字段。

## 5. 组件栏与画布交互

### 5.1 组件栏信息架构

画布左侧组件栏纵向分为五个区域，并支持隐藏滚动条的上下滚动：

1. 流程组件：分组组件、关联组件、结束节点；
2. 文本组件：六张细粒度文本卡片；
3. 数值组件：三张细粒度数值卡片；
4. 日期组件：日期转换；
5. 规则组件：空值填充、类型转换、条件映射。

每张卡片显示四字名称和一句用途说明，不使用只有技术函数名的文案。同一分类使用一致的边框、底色、图标底色和画布节点色；颜色同时延伸到配置抽屉和发布版本拓扑预览。

### 5.2 新增组件

- 点击卡片：在当前视口可见区域的空白位置新增组件并自动选中；
- 拖拽卡片：在用户释放的位置新增组件；
- 新增组件不自动连接任何上游，必须由用户明确拖线；
- 画布组件文字不可选中；点击组件任意非操作按钮区域打开配置，右半区为拖出连线热区，左半区为接收连线热区，边缘圆点只用于提示方向而不是要求精确命中；
- 连线采用独立指针手势，不与整卡移动抢占拖拽；拖到空白处松开只取消本次连线，不移动组件也不改变已有连接；
- 从已有组件输出端口拖到字段处理卡片或空白画布：后续可作为快捷创建增强项，P0 不强制；
- 新组件没有输入时显示“请连接上游产物”，配置抽屉只允许填写名称，字段选择和规则配置保持禁用；
- 接入输入后，组件立即读取上游稳定产物列表并开放配置。

### 5.3 组件卡片状态

卡片至少展示：

- 组件类型和用户命名；
- 已配置规则数；
- 新增字段数、替换字段数；
- 输入字段数和输出字段数；
- 未配置、配置有效、存在警告或配置错误状态；
- 输出端口。

规则摘要最多展示两条，例如“`order_time` 转标准日期”“`order_month` 提取月份”；更多规则显示“另有 3 条”。

### 5.4 配置抽屉结构

配置抽屉沿用现有关联和分组组件的交互层级：

1. 顶部：组件名称、上游输入，只读展示输入来源；
2. 中部：规则列表，支持新增、复制、删除和调整顺序；
3. 规则编辑区：输入字段、处理方式、参数、输出字段和失败策略；
4. 底部：转换前后样例与质量摘要；
5. 操作区：取消、完成。

用户点击“完成”时进行客户端快速校验；关闭抽屉不代表保存数据集，数据集保存仍复用现有服务端 DSL 校验和乐观锁流程。

## 6. 通用规则配置合同

每条规则必须包含：

| 配置项 | 规则 |
|---|---|
| 规则名称 | 自动生成，可编辑，最长 100 字符 |
| 输入参数 | 一个或多个上游字段，也可以包含受控字面量 |
| 处理方式 | 只能选择当前 `family` 允许的白名单函数 |
| 函数参数 | 根据处理方式动态展示并严格校验 |
| 输出名称 | 必填，默认基于来源业务名称生成 |
| 输出编码 | 必填、稳定、符合 DSL identifier 规则、当前组件产物内唯一 |
| 输出类型 | 根据函数推导；只有语义明确时才允许用户选择 |
| 输出方式 | “新增字段”或“替换来源字段”，默认新增 |
| 失败策略 | 阻止运行、输出 NULL、使用默认值、保留原值 |
| 说明 | 可选，描述业务用途，不参与执行 |

### 6.1 失败策略

并非所有函数都支持全部失败策略：

| 策略 | 说明 | 适用范围 |
|---|---|---|
| 阻止运行 | 任一非空输入无法转换时返回结构化错误 | 默认策略；所有可能失败的函数 |
| 输出 NULL | 失败值输出 NULL，并计入失败数 | 类型、日期、文本提取、数值计算 |
| 使用默认值 | 失败时输出用户配置且类型合法的固定值 | 类型、日期、条件、空值、数值 |
| 保留原值 | 失败时返回原始值，要求输入输出类型兼容 | 文本清理及同类型转换 |

“输出 NULL”只处理执行失败，不等同于把正常输入 NULL 转换为错误。输入 NULL 的行为由具体函数定义，并在配置区明确显示。

### 6.2 输出字段身份

- 每个输出字段拥有不可变的 `outputId`；
- 名称和编码允许在草稿中修改，但修改必须经过下游引用影响检查；
- 下游组件引用 `outputId` 或等价稳定身份，不能只依赖可编辑名称；
- 对外 DSL 字段编码在同一产物内必须唯一；
- 复制规则时生成新的规则 ID 和输出字段 ID，并对编码自动追加可编辑后缀；
- 删除被下游使用的规则前展示影响组件，并要求用户确认或先修复引用。

### 6.3 类型驱动

- 输入字段列表按函数接受的数据类型过滤；
- 用户切换函数后，如果现有输入不再合法，应清空该输入并说明原因；
- 输出类型由函数推导，不能由用户随意声明与实际结果不一致的类型；
- `INTEGER`、`DECIMAL`、`STRING`、`BOOLEAN`、`DATE` 和 `DATETIME` 使用平台规范类型；
- 跨源执行必须先按平台规范类型归一化，再应用同一转换语义。

## 7. 七类组件详细规格

### 7.1 日期处理

#### P0 函数

| 功能 | 输入 | 参数 | 输出 |
|---|---|---|---|
| 解析日期 | STRING | 输入格式、时区、严格模式 | DATE 或 DATETIME |
| 格式化日期 | DATE/DATETIME | 输出格式、时区 | STRING |
| 日期归整 | DATE/DATETIME | DAY/WEEK/MONTH/QUARTER/YEAR | DATE 或 DATETIME |
| 提取日期部分 | DATE/DATETIME | YEAR/QUARTER/MONTH/WEEK/DAY/WEEKDAY/HOUR/MINUTE/SECOND | INTEGER |
| 日期加减 | DATE/DATETIME | 数量、单位 | DATE 或 DATETIME |
| 日期差 | 两个 DATE/DATETIME | DAY/HOUR/MINUTE/SECOND | INTEGER 或 DECIMAL |
| 时区转换 | DATETIME | 来源时区、目标时区、歧义时间策略 | DATETIME |

#### 交互规则

- 解析日期提供常用格式选项和“自定义格式”，不要求用户输入数据库方言格式；
- 系统将产品级日期格式转换为各执行器语义，MySQL、Oracle 和文件执行器必须一致；
- 周归整必须明确采用 ISO 周，周一为一周起点；
- 日期差必须明确方向，界面显示“结束日期 - 开始日期”；
- 时区缺失时使用数据集声明时区，不能静默使用 API 服务器本地时区；
- 夏令时不存在或重复的本地时间必须按配置失败，不能由不同数据库自行选择。

#### 示例

`2026/07/21 13:30:00` 按 `yyyy/MM/dd HH:mm:ss` 解析为 DATETIME，再归整到 MONTH，输出 `2026-07-01 00:00:00`。

### 7.2 文本处理

#### P0 函数

| 功能 | 参数 | 说明 |
|---|---|---|
| 按位置截取 | 起始位置、长度 | 产品统一使用从 1 开始的位置 |
| 左侧/右侧截取 | 字符数 | 按 Unicode 字符而非字节计数 |
| 分隔符前后截取 | 分隔符、第几次出现、前/后 | 未找到分隔符按失败策略处理 |
| 去除空白 | 左侧/右侧/两侧 | 空白字符集合由平台定义 |
| 大小写转换 | 大写/小写 | 首期采用 locale-neutral 规则 |
| 文本替换 | 查找文本、替换文本、首次/全部 | P0 为普通文本替换 |
| 正则提取 | RE2 模式、捕获组 | 限制模式长度和捕获组数量 |
| 文本长度 | 无 | 返回 Unicode 字符数 |

#### 交互规则

- 文本函数通过“大写转换、空格清理、文本替换、小写转换、字段截取、字段拼接”独立卡片进入；单功能卡片的处理逻辑只读，避免用户再次从大列表中寻找；
- 配置区使用真实样例辅助用户确认起始位置和长度；
- 正则采用 RE2 兼容子集，避免灾难性回溯；
- 空字符串是有效字符串，不与 NULL 混同；如需统一必须显式配置空值处理；
- 超出字符串长度的截取返回可用部分，起始位置越界返回空字符串，不视为系统错误。

#### 示例

订单编码 `CN-2026-000123` 以 `-` 拆分或截取，输出国家编码 `CN` 和订单序号 `000123`。

### 7.3 类型转换

#### P0 函数

- STRING → INTEGER/DECIMAL/BOOLEAN/DATE/DATETIME；
- INTEGER/DECIMAL/BOOLEAN/DATE/DATETIME → STRING；
- INTEGER ↔ DECIMAL；
- DATE ↔ DATETIME；
- INTEGER/STRING → BOOLEAN，必须配置真值和假值映射。

#### 交互规则

- 字符串转数值可配置小数点符号和千分位符号，但 P0 不做自动语言猜测；
- DECIMAL 转 INTEGER 必须选择截断或四舍五入，不能由数据库隐式决定；
- 字符串转日期复用日期处理的解析规则；
- 转字符串生成语义字符串，不承担报表展示格式；
- 溢出、非法字符、NaN 和无穷值必须进入失败策略；
- Oracle 没有原生 BOOLEAN 列类型时仍返回平台 BOOLEAN 语义，不能把数据库实现细节暴露给用户。

### 7.4 字段计算

#### P0 函数

| 功能 | 输入/参数 |
|---|---|
| 加、减、乘、除 | 两个或多个数值字段/字面量 |
| 比例 | 分子、分母、比例因子，默认 1 |
| 增长率 | 当前值、基准值，结果为 `(当前-基准)/基准` |
| 绝对值 | 一个数值字段 |
| 四舍五入 | 数值字段、小数位数，模式固定 HALF_UP |
| 向上/向下取整 | 一个数值字段 |
| 范围限制 | 数值字段、最小值、最大值 |
| 单位换算 | 数值字段、乘数、除数 |

#### 交互规则

- 除法、比例和增长率必须配置除零策略，默认阻止运行；
- 计算表达式用结构化步骤配置，不允许输入自由 SQL；
- 输出优先使用 DECIMAL，只有可以证明不会产生小数和溢出时才推导 INTEGER；
- P0 固定 HALF_UP 舍入，避免数据库默认舍入规则漂移；
- UI 显示可读公式摘要，例如“`amount_cent ÷ 100 → amount_yuan`”。

### 7.5 条件映射

#### P0 模式

1. 精确值映射：`A → 有效`、`I → 无效`；
2. 范围分档：金额 `<100`、`100–999`、`≥1000`；
3. 多条件 CASE：支持 AND、OR、等于、不等于、比较、包含、在…中、为空和非空；
4. 其他值：配置 ELSE 输出或 NULL。

#### 交互规则

- 规则自上而下匹配，命中第一条后停止；
- 拖动排序后立即刷新样例预览，并提示结果可能变化；
- 所有分支输出必须可统一为同一规范类型；
- “在…中”保存为结构化数组，每个候选项独立声明为自定义值或上游字段；执行 DSL 使用 `IN(left, ARRAY(...))`，不拼接自由文本 SQL；
- 精确值映射支持 CSV 式批量粘贴，但解析后仍保存为结构化映射，不保存自由文本；
- 范围必须检查重叠和空洞：重叠阻止完成，空洞在存在 ELSE 时允许；
- 映射项较多时支持搜索，P0 单条规则最多 500 个精确映射项。

### 7.6 空值处理

#### P0 函数

- NULL 填充固定默认值；
- 空字符串转 NULL；
- 仅包含空白的字符串转 NULL；
- 指定占位值转 NULL，例如 `N/A`、`-`；
- 多字段依次取第一个非 NULL 值；
- 主字段为空时取备用字段；
- 可选：数值异常占位值转 NULL，但必须配置精确值。

#### 交互规则

- NULL、空字符串、仅空白字符串和字符串 `"null"` 默认是四种不同值；
- 多字段依次取值时，所有候选字段必须类型兼容；
- 默认值必须通过目标类型校验；
- 空值处理不能把未授权字段值复制到授权级别更低的输出字段；
- 配置区显示“输入空值数”和“处理后空值数”，不展示未经授权的原始异常样例。

### 7.7 字段拆分与合并

#### P0 函数

| 功能 | 参数 | 输出 |
|---|---|---|
| 按分隔符拆分 | 分隔符、最多拆分数、是否保留空段 | 多个 STRING 字段 |
| 按固定位置拆分 | 一个或多个切分位置 | 多个 STRING 字段 |
| 取第 N 段 | 分隔符、段序号 | 一个 STRING 字段 |
| 多字段拼接 | 字段/字面量列表、分隔符 | 一个 STRING 字段 |
| 条件拼接 | 是否忽略 NULL、是否忽略空字符串 | 一个 STRING 字段 |

#### 交互规则

- 多输出规则必须一次配置所有输出名称和编码；
- 样例预览根据真实输入提示建议输出列数，但不能根据样例静默截断正式数据；
- 实际段数超过配置上限时按失败策略处理；
- 拼接中的 NULL 行为必须显式选择：结果为 NULL、忽略 NULL 或把 NULL 替换为固定文本；
- 固定位置继续采用从 1 开始的 Unicode 字符位置。

## 8. 转换预览与质量反馈

### 8.1 预览方式

- 配置抽屉使用受控上游预览，默认最多 20 行；
- 左侧展示参与规则的输入字段，右侧展示输出字段；
- 替换字段同时展示“转换前”和“转换后”；
- 敏感字段继续遵守列策略和脱敏规则，不能为了配置转换返回未脱敏样例；
- 预览请求必须携带草稿版本、画布指纹或等价代次，迟到响应不能覆盖新配置。

### 8.2 质量摘要

每条规则返回：

- 样例总行数；
- 输入 NULL 数；
- 转换成功数；
- 转换失败数；
- 输出 NULL 数；
- 使用默认值数；
- 截断或溢出警告数。

错误只返回规则 ID、字段 ID、错误码、计数和脱敏原因，不在审计或错误消息中保存失败原值。

### 8.3 完成与保存门禁

- 规则配置不完整时阻止关闭抽屉，并定位第一条错误；
- “阻止运行”策略在样例中出现失败时允许保存草稿，但显示高风险警告；
- 服务端 DSL 结构或类型校验失败时阻止保存；
- 发布审批的正式试跑只要遇到“阻止运行”错误就拒绝发布；
- 样例无错误不等于正式数据无错误，界面不得显示“全部数据转换成功”。

## 9. DSL 与设计器合同

### 9.1 设计器图扩展

建议扩展：

```text
GraphInput.kind = NODE | JOIN | GROUP | TRANSFORM
CanvasComponentKind = NODE | JOIN | GROUP | TRANSFORM | END
DesignerGraphV1.transforms = GraphTransform[]
```

`GraphTransform` 至少包括：

```text
id
name
family
input
position
rules[]
```

规则至少包括：

```text
id
operation
arguments[]
options
outputs[]
onError
```

`arguments` 只能是上游产物字段引用或受控字面量；`options` 必须由不同 `operation` 的严格 Schema 约束，不能成为任意 JSON 参数袋。

### 9.2 DSL 表达方式

转换组件属于设计器拓扑，最终执行 DSL 继续以结构化字段表达式为权威事实，不保存 SQL。转换产物字段表达式可以递归引用来源字段或上游转换字段，但规范化后必须具有确定性顺序、深度和节点预算。

`TRANSFORM` 不能只保存在 `designer` 展示元数据中，也不能只把表达式附着到最终 `fields[]`。最终字段表达式无法单独表达“先转换，再使用转换结果进行 Join、分组或另一次转换”的执行顺序。

技术设计必须选择并正式固化以下等价方案之一：

1. 在执行 DSL 中增加拓扑化 `transforms[]`，每项保存输入组件引用、产物字段及表达式，下游 Join、分组和结束节点可以引用转换产物；
2. 引入明确的中间派生字段引用，例如 `DERIVED_FIELD_REF(transformId, outputId)`，由规范计划将其解析成有序的 `PROJECT/TRANSFORM` 逻辑算子；
3. 使用经过证明的确定性表达式内联，但必须同时支持 Join 条件、分组维度、聚合参数、过滤和最终输出，并保持中间字段血缘、错误路径及表达式预算。

P0 推荐前两种方案。无论选择哪一种，规范逻辑计划都必须显式出现 `TRANSFORM` 或等价 `PROJECT` 算子，并位于画布声明的上游与下游算子之间。仅在前端生成预览值、保存 designer 配置或修改最终输出表达式，不视为完成转换组件。

现有表达式可直接复用：

| 组件 | 已有表达式 |
|---|---|
| 日期处理 | `DATE_TRUNC` |
| 类型转换 | `CAST` |
| 字段计算 | `ADD`、`SUBTRACT`、`MULTIPLY`、`DIVIDE` |
| 条件映射 | `CASE`、`IN`、`ARRAY` 及比较/逻辑表达式 |
| 空值处理 | `COALESCE`、`IS_NULL`、`IS_NOT_NULL` |
| 字段拆分与合并 | `CONCAT` |

P0 需要新增或正式化的表达式语义：

| 领域 | 建议表达式 |
|---|---|
| 日期 | `DATE_PARSE`、`DATE_FORMAT`、`DATE_EXTRACT`、`DATE_ADD`、`DATE_DIFF`、`TIMEZONE_CONVERT` |
| 文本 | `SUBSTRING`、`TRIM`、`UPPER`、`LOWER`、`REPLACE`、`REGEXP_EXTRACT`、`LENGTH` |
| 数值 | `ABS`、`ROUND`、`CEIL`、`FLOOR`、`CLAMP` |
| 空值 | `NULL_IF` |
| 拆分 | `SPLIT_PART`，多输出拆分由多个确定性表达式共同描述 |

实施进展（2026-07-21）：文本领域的 `SUBSTRING`、`TRIM`、`UPPER`、`LOWER`、`REPLACE` 已完成前端配置、DSL 校验、MySQL/Oracle 安全编译、文件求值和真实数据库金丝雀；`GROUP → TRANSFORM` 已保持隐藏分组粒度并支持聚合结果外层转换；本表其余表达式仍按阶段 B 推进。

命名可以在技术设计阶段调整，但每个表达式必须有跨方言、文件执行、NULL、类型、失败和时区的正式语义，不能直接映射为某个数据库的私有函数名称。

### 9.3 回载兼容

- 没有 `transforms` 的历史 `DesignerGraphV1` 按空数组处理；
- 新草稿保存后必须能从 DSL 与 designer 信息完整恢复转换组件、位置、规则和字段身份；
- 已发布版本保持不可变，不回写新 designer 结构；
- 旧客户端读取含 `TRANSFORM` 的草稿时必须失败关闭或进入只读兼容模式，不能忽略转换后继续保存；
- Schema 版本升级需要迁移器和确定性哈希回归测试。

## 10. 执行语义与安全边界

### 10.1 跨执行器一致性

每个新增表达式必须同时实现并测试：

1. 数据集领域校验器；
2. JSON Schema；
3. MySQL 安全编译器；
4. Oracle 安全编译器；
5. Excel/CSV 文件求值器；
6. 跨源网关规范类型路径；
7. 规范化、哈希、血缘索引和发布试跑。

任何路径尚未实现时，该数据源组合必须返回稳定的“不支持转换”错误，不能静默跳过或回退成不同语义。

失败策略属于执行语义的一部分。数据库编译器不能在方言行为不一致时直接使用裸 `CAST`、日期函数或隐式类型转换；必须通过受控判断、CASE 或等价安全算子实现与文件求值器一致的“阻止运行、输出 NULL、默认值、保留原值”结果。

### 10.2 权限与敏感字段

- 字段选择列表只显示调用者有权读取且允许参与表达式的字段；
- 输出字段继承所有输入字段中最严格的敏感级别和列策略，除非由受信任策略系统明确降级；
- 用户配置“替换来源字段”不能清除来源字段策略；
- 脱敏、哈希和部分隐藏属于列策略能力，不作为普通转换函数开放；
- 查询审计保存表达式计划哈希、规则 ID、执行状态和计数，不保存字段值、正则命中内容或映射前原值。

### 10.3 资源限制建议

P0 建议采用以下上限，最终数值由性能测试确认：

- 单数据集最多 32 个字段处理组件；
- 单组件最多 20 条规则；
- 单组件最多产生 32 个新字段；
- 单数据集最多产生 128 个转换字段；
- 单条规则最多 8 个输入参数；
- 正则模式最长 256 字符，捕获组最多 10 个；
- 精确值映射最多 500 项；
- 表达式最大递归深度和节点总数继续受服务端统一预算限制；
- 字符串结果必须设置最大长度，超过上限按失败策略处理。

客户端限制只用于即时反馈，服务端必须重复校验。

### 11.2 必测边界

- NULL、空字符串、全空白、非 ASCII、中文、Emoji 和组合 Unicode 字符；
- 超长字符串、截取越界、分隔符缺失、多余分段和正则无匹配；
- 整数溢出、小数精度、负数、除零、NaN 和无穷值；
- 闰年、月末、季度边界、ISO 跨年周、时区偏移和夏令时重复/缺失时间；
- CASE 重叠、无 ELSE、分支类型不一致和批量映射重复键；
- 删除来源字段、删除转换规则、修改输出编码和下游组件占用；
- 保存版本冲突、预览取消、超时、迟到响应和重复请求；
- 未授权字段、脱敏字段、最小分组策略和跨租户引用；
- 大量规则、深层串联和接近预算上限的表达式。

### 11.3 跨方言金丝雀样例

建立一组不含敏感数据的固定样例，在 MySQL、Oracle 和文件执行器上执行同一 DSL，并逐字段比较规范结果。日期时间比较必须基于明确时区和规范序列化，DECIMAL 不能通过浮点近似掩盖差异。

## 12. 开发拆分建议
